# План реализации сайта-портфолио

## Обзор проекта

Создание сайта-портфолио для дочери с простым управлением контентом через iPhone.

**Технологический стек:**
- **Генератор**: Jekyll (Ruby-based статический генератор)
- **Хостинг**: GitHub Pages
- **CMS**: Decap CMS (бывший Netlify CMS)
- **Хранилище фото**: GitHub репозиторий
- **Стиль**: Тёмная тема, Masonry-сетка, минимализм (по референсу peterbeavis.com)

---

## Этап 1: Инициализация проекта

### Задачи 1.1: Создание структуры репозитория

```
portfolio/
├── _config.yml           # Конфигурация Jekyll
├── _includes/            # Переиспользуемые компоненты
├── _layouts/             # Шаблоны страниц
├── _posts/               # Альбомы/сессии (как посты Jekyll)
├── _albums/              # Данные об альбомах (YAML)
├── assets/
│   ├── css/              # Стили
│   ├── js/               # Скрипты (Masonry-сетка)
│   └── images/           # Фото для альбомов
├── admin/                # Конфигурация Decap CMS
│   └── config.yml
├── index.html            # Главная страница
├── contact.html          # Контактная информация
├── about.md              # О фотографе
└── README.md             # Документация
```

### Задачи 1.2: Настройка Jekyll

**_config.yml:**
```yaml
title: "Имя дочери - Photography Portfolio"
description: "Photography portfolio"
baseurl: ""
url: ""

# Коллекции для альбомов
collections:
  albums:
    output: true
    permalink: /albums/:name/

# Настройки сборки для GitHub Pages
markdown: kramdown
theme: minima
```

## Этап 3: Decap CMS интеграция

### Задачи 3.1: Настройка админ-панели

**admin/config.yml:**
```yaml
backend:
  name: git-gateway
  branch: main

media_folder: "assets/images"
public_folder: "/assets/images"

collections:
  - name: "albums"
    label: "Альбомы"
    folder: "_albums"
    create: true
    slug: "{{slug}}"
    fields:
      - {label: "Название", name: "title", widget: "string"}
      - {label: "Описание", name: "description", widget: "text", required: false}
      - {label: "Обложка", name: "cover", widget: "image"}
      - {label: "Фотографии", name: "photos", widget: "list", field: {widget: "image"}}
      - {label: "Раздел", name: "category", widget: "select", options: ["Люди", "Собаки", "Природа", "Другое"]}
```

## Этап 4: Развёртывание

### Задачи 4.1: GitHub Pages настройка

1. Создать репозиторий `portfolio`
2. Включить GitHub Pages в Settings → Pages
3. Выбрать источник: GitHub Actions (для поддержки плагинов)

**.github/workflows/jekyll.yml:**
```yaml
name: Deploy Jekyll site
on:
  push:
    branches: [main]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: ruby/setup-ruby@v1
        with:
          ruby-version: '3.1'
          bundler-cache: true
      - run: bundle install
      - run: bundle exec jekyll build
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./_site
```

### Задачи 4.2: Покупка и привязка домена (позже)

1. Купить домен
2. Добавить CNAME запись в репозиторий
3. Настроить DNS у регистратора
